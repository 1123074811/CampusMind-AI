# Roadmap

本文件记录 CampusMind AI 智能体能力的实施状态与后续拆分。

## 校园事件智能行动助手

### 功能定位

校园事件智能行动助手面向学生端和管理端之间的最后一公里：不只展示校园通知，而是把爬虫和智能体清洗后的事件转换为用户可以直接执行的行动卡。

当前已进入 R6 第一阶段：认知 Agent 可从完整信息中生成精简卡片字段，并保留原始信息 ID 和原文链接。自动批处理、持久化和学生端卡片展示后续接入。

### 第一阶段已实现

- 复用现有认知 Agent，不新增重复的 Agent 层级。
- `POST /api/v1/ai/cognition/extract` 支持 `originalItemId`、`originalUrl` 和完整正文。
- 通用输出包含摘要、关键时间、面向对象和所需动作。
- 竞赛输出包含报名开始时间、截止时间、持续时间、所需材料、报名网址、参赛方式、组队要求和附件。
- 默认规则模式可独立运行；LLM 模式使用结构化输出，失败时自动降级规则抽取。
- 缺失字段返回空值，不影响原文展示，不允许模型编造原文链接。

### 目标价值

- 降低学生遗漏报名、提交材料、考试调整、课程变更等关键信息的风险。
- 把通知原文转为清晰的下一步动作，减少用户自行阅读和归纳的成本。
- 让系统从“校园信息聚合”升级为“校园行动规划”。

### 预期能力

- 自动识别事件是否和当前用户相关，例如年级、专业、课程、身份、订阅范围。
- 自动生成行动卡，包括截止时间、地点、所需材料、原文链接、建议操作。
- 自动生成提醒计划，例如报名截止前 3 天、1 天、2 小时提醒。
- 自动跟踪原网页变化，例如截止时间变更、附件更新、活动取消。
- 自动解释推荐理由，例如“与你的专业相关”“你订阅了创新创业通知”。

### 行动卡示例

```text
你可能需要报名

事件：创新创业竞赛报名开放
截止：7 月 15 日 18:00
所需材料：报名表、项目计划书、学生证照片
建议动作：今天先完成报名表，7 月 13 日前提交计划书初稿
原文链接：https://...
推荐原因：面向全校学生，且你订阅了竞赛类事件
```

### 数据预留

后续可在现有事件模型基础上补充以下结构：

- `action_required`：是否需要用户行动。
- `action_type`：报名、提交材料、到场、考试确认、课程调整确认等。
- `deadline_at`：最终截止时间。
- `reminder_plan`：提醒时间点列表。
- `required_materials`：所需材料列表。
- `source_revision_hash`：原文变更检测哈希。
- `recommend_reason`：推荐/提醒原因。

### 服务预留

建议后续新增或扩展以下能力：

- 在 `campus-ai-service` 中新增行动抽取 Agent，把结构化事件转为行动建议。
- 在 `campus-feed-service` 中按用户画像筛选行动卡。
- 在 `campus-user-service` 中维护提醒偏好、订阅范围和已完成状态。
- 在 Flutter 用户端增加行动卡列表、提醒开关、完成状态和原文入口。

### 下一阶段

- 不在当前后台页面实现行动卡 UI。
- 不新增提醒推送任务。
- 不新增数据库表迁移。
- 将认知 Agent 接入 `information_item` 自动批处理与结果持久化后，再改变学生端展示主链路。
